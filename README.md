# Friendship Points API

Friendship Points API is a TypeScript backend for tracking friendship-related events, rules, notes, predictions, and point changes over time.

It is primarily a learning project for practising backend engineering, TypeScript, PostgreSQL, Prisma, Fastify, LangChain, structured LLM outputs, testing, linting, CI, error handling, RAG, observability, deployment, security, and eventually a simple responsive frontend.

The core idea is that each friendship has a running point balance. Events can increase or decrease that balance. For example, a thoughtful action might add points, while an action that violates a known preference might subtract points.

Assessments can be added manually or generated by an LLM. When the LLM judges an event, it looks at the event text, the friend’s notes, and active friendship rules. It then returns a structured judgement with a score change, reasoning, impact direction, confidence, matched rules, and bias notes. The backend stores the result as an Assessment, and the balance endpoint calculates the current friendship score by summing all assessment scoreDelta values for that friend’s events.

Predictions work similarly, but they are hypothetical. A prediction asks, “What would happen if I did this?” The API sends the hypothetical action to the LLM with the relevant friend context, but it does not save a new event or assessment.

This project is meant to be social commentary on the social credit score imposed by the CCP

## What the app does

The API lets you:

- create and search friends
- define friendship rules and preferences
- record friendship-related events
- manually assess events with point changes
- generate mock and real LLM assessments
- generate mock and real Mistral predictions for hypothetical actions
- track friendship balances
- update friend information
- append friend notes
- soft-delete friends without immediately removing database history
- track a friendship point balance based on stored assessments
- let an LLM judge events using rules, notes, and event text

Example:

```txt
Rule:
Cole dislikes unexpected phone calls.

Event:
I called Cole without warning.

Possible assessment:
This may negatively affect the friendship because it violates a known preference.
```
